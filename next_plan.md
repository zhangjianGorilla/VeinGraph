# VeinGraph 未来扩展路线图

---

## 🔥 第一梯队：高价值扩展（马上就能落地，效果炸裂）

| 方向 | 说明 |
|------|------|
| **多模态文档支持** | 当前只支持纯文本 PDF/Word。可以接入 OCR（如 PaddleOCR）处理扫描件，接入表格解析引擎识别 Excel/CSV，甚至用视觉大模型（GPT-4o Vision）直接"看"图片里的组织关系图 |
| **图谱社区发现** | 在 Neo4j 里跑 Louvain / Label Propagation 等图算法，自动识别出紧密关联的实体群落。比如上传公司架构文档后，它能自动告诉你"这几个人属于财务部门的核心关系网" |
| **多跳推理问答** | 当前的 Text2Cypher 只做了单跳查询。升级为多跳链式推理后，用户可以问"A 认识的人里，谁又和 C 有过合作？"——真正的六度空间搜索 |
| **用户认证与权限** | 接入 Spring Security + JWT / OAuth2，支持多用户登录、文档权限隔离。这对开源社区来说是刚需 |

---

## ⚡ 第二梯队：差异化竞争力（做了就和市面上的项目拉开巨大差距）

| 方向 | 说明 |
|------|------|
| **知识图谱时间线** | 给每条关系加上时间维度。前端可以拖动时间滑块，看到"2024年1月，A和B建立合作→3月，B和C产生矛盾→6月，A退出"的动态演化过程 |
| **人工纠错反馈环** | 前端图谱节点支持右键菜单："这条关系是错的"——标记后回写 MongoDB，作为 Few-Shot 负样本反馈给 LLM，形成飞轮式的抽取质量提升 |
| **增量更新与版本快照** | 当前是全量重建图谱。改为增量 Diff——同一份文档修改后重新上传，系统自动计算新增/删除/变更的实体与关系，并在 Neo4j 中原子化地打补丁 |
| **跨文档关系融合** | 目前图谱是按文档隔离的。允许用户选择多篇文档进行"全局合并"，自动发现跨文档的潜在关联（比如文档A的"王建国"和文档B的"王局长"是否同人） |

---

## 🌐 第三梯队：生态集成（让项目变成平台级产品）

| 方向 | 说明 |
|------|------|
| **Webhook 通知** | 文档处理完成后自动推送飞书/钉钉/Slack 通知 |
| **数据导出** | 支持将图谱导出为 GraphML、JSON-LD、CSV 格式，方便导入 Gephi 等专业分析工具 |
| **插件化抽取器** | 定义 SPI 接口，允许社区贡献自定义的领域抽取器（如：医疗报告抽取器、法律判决书抽取器） |
| **外部知识库对齐** | 将抽取出来的实体对齐到 Wikidata / 百度百科，补充额外属性（出生日期、职位、公司等） |
| **移动端适配** | 前端响应式改造，支持 iPad / 手机上浏览图谱和进行问答 |

---

## 🧪 彩蛋方向（学术味浓，发论文专用）

- **自动知识图谱补全**：用 TransE/RotatE 等图嵌入模型预测缺失的关系
- **对比式 RAG A/B 测试**：同时展示"纯向量 RAG"和"GraphRAG"的回答，让用户直观感受差距
- **联邦图谱**：多个 VeinGraph 实例之间安全地共享部分图谱子图，不暴露原始文档
